InkLink provides a **hosted** flow solution that offers the simplest integration path, with InkLink handling the entire request flow.

## Benefits

- **Zero Frontend Development** - No SDK or UI components needed
- **Minimal Integration Complexity** - Simple redirect-based flow
- **No SDK Overhead** - Keeps your application lightweight
- **Automatic Updates** - Always get the latest InkLink features

## Creating a Request

Requests define the structure and requirements of your data verification flow. You can create requests through our UI builder or programmatically via API. In return, you will receive a `request_url` that you can share with users to verify their data.

## Creating Request via UI

Our Proof Request Builder UI available in the InkLink WebApp provides an interface for creating requests without code.

### Request Builder Sections

| Setting | Description | Best Practice |
| ----- | ------- | ------------------------- |

## Creating Request via API

For programmatic request creation, use the following endpoint:

**Request**

<CodeGroup>
```shell curl
curl https://app.inklink.com/api/request \
    --header "x-api-key: $secret_test_xxx" \
    --header "content-type: application/json" \
    --data \
'{
    "proof_request_description": "Verify Tesla ownership",
    "proof_title": "Vehicle Ownership Verification",
    "category": "OWNERSHIP",
    "return_url": "https://your-app.com/request-complete"
}'
```

```python Python
import requests

url = "https://app.inklink.com/api/request"

headers = {
    "x-api-key": "secret_test_xxx",
    "Content-Type": "application/json"
}

data = {
    "proof_request_description": "Verify Tesla ownership",
    "proof_title": "Vehicle Ownership Verification",
    "category": "OWNERSHIP",
    "return_url": "https://your-app.com/request-complete"
}

response = requests.post(url, headers=headers, json=data)

print(response.json())
```

```typescript TypeScript
const response = await fetch("https://app.inklink.com/api/request", {
  method: "POST",
  headers: {
    "x-api-key": "secret_test_xxx",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    proof_request_description: "Verify Tesla ownership",
    proof_title: "Vehicle Ownership Verification",
    category: "OWNERSHIP",
    return_url: "https://your-app.com/request-complete",
  }),
});

const data = await response.json();
console.log(data);
```

</CodeGroup>

**Minimal Request**
You can pass in optional fields to the request payload outlined [here](/api-reference/endpoint/create) {/* link to request page */}

```json Required request body field
{
  "proof_request_description": "Verify account ownership"
}
```

**Response**

```json JSON
{
  "request_id": "req_xxx",
  "request_url": "https://app.inklink.com/request/req_xxx"
}
```

## Tracking Request Session

If you need to track a specific user's request status, use the following endpoint to create a unique `session_id`:

**Request**

<CodeGroup>

```shell curl
curl https://app.inklink.com/api/request-session-id \
    --header "x-api-key: $secret_test_xxx" \
    --header "content-type: application/json" \
    --data \
'{
  "request_id": "req_xxx",
  "return_url": "https://your-app.com/request-complete?your-ref-id=xxx"
}'
```

```python Python
import requests

url = "https://app.inklink.com/api/request-session-id"

headers = {
    "x-api-key": "secret_test_xxx",
    "Content-Type": "application/json",
}

data = {
    "request_id": "req_xxx",
    "return_url": "https://your-app.com/request-complete?your-ref-id=xxx",
}

response = requests.post(url, headers=headers, json=data)

print(response.json())

```

```typescript TypeScript
const response = await fetch("https://app.inklink.com/api/request-session-id", {
  method: "POST",
  headers: {
    "x-api-key": "secret_test_xxx",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    request_id: "req_xxx",
    return_url: "https://your-app.com/request-complete?your-ref-id=xxx",
  }),
});

const data = await response.json();
console.log(data);
```

</CodeGroup>

**Response**

```json JSON
{
  "session_id": "sid_xxx",
  "request_url": "https://app.inklink.com/request/req_xxx?session_id=sid_xxx"
}
```
