---
title: "Retrieving Results"
---

After a user completes a request, retrieve the results using the `request_id` and optionally a `session_id` (if you want results from a specific session).

To retrieve request results, use the following API:

**Request**

<CodeGroup>

```shell curl
curl https://app.inklink.com/api/request-result/req_xxx?session_id=sid_xxx \
    --header "x-api-key: $secret_test_xxx"
```

```python Python
import requests

url = "https://app.inklink.com/api/request-result/req_xxx?session_id=sid_xxx"

headers = {
    "x-api-key": "secret_test_xxx",
}

response = requests.get(url, headers=headers)

print(response.json())
```

```typescript TypeScript
const response = await fetch(
  "https://app.inklink.com/api/request-result/req_xxx?session_id=sid_xxx",
  {
    method: "GET",
    headers: {
      "x-api-key": "secret_test_xxx",
    },
  }
);

const data = await response.json();
console.log(data);
```

</CodeGroup>

**Response**

```json JSON
{
  "status": "PASSED",
  "claims": [
    {
      "property": "account_holder_name",
      "value": "John Doe"
    },
    {
      "property": "account_balance",
      "value": "$12,345.67"
    }
  ],
  "acceptance_level": "HIGH",
  "acceptance_reason": "All validation criteria met",
  "credibility_level": "HIGH",
  "credibility_reason": "Clear screenshot with matching user data"
}
```

The claims AI-generated, they will change with each request

**Status Values**

| Status   | Description                         |
| -------- | ----------------------------------- |
| `PASSED` | Verification completed successfully |

**Credibility Levels**
| Level | Color | Description | Recommended Action |
| - | - |-|-|
| `HIGH` | Verification completed successfully | Ready for auto-approval | Accept automatically |
| `MEDIUM` | Verification failed validation | Minor issues detected | Consider spot-check |
| `LOW` | Verification in progress | Significant Issues | Manual review required |
