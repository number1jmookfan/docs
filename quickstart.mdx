## Step 1: Send an Http Request

Define the data you need to verify

```shell Create a simple proof request
curl --location 'https://app.inklink.com/api/request' \\
--header 'x-api-key: secret_test_xxxx' \\
--header 'Content-Type: application/json' \\
--data '{
"proof_request_description": "Verify Tesla ownership",
"proof_title": "Vehicle Ownership Verification",
"category": "OWNERSHIP",
"return_url": "https://your-app.com/request-complete"
}'
```

### Response

The response contains the URL given to users to complete the data verification process

```json Http response
{
  "request_id": "req_xxx",
  "request_url": "https://app.inklink.com/request/req_xxxx"
}
```

### (Optional) Generate a Session Id

You can create a unique `session_id` to track when a specific user completes their request.

```shell
curl --location 'https://app.inklink.com/api/request-session-id' \\
--header 'x-api-key: secret_test_xxxx' \\
--header 'Content-Type: application/json' \\
--data '{
  "request_id": "req_xxxx",
  "return_url": "https://your-app.com/request-complete?your-ref-id=xxx"
}'
```

### Response

```json
{
  "session_id": "sid_xxx",
  "request_url": "https://app.inklink.com/request/req_xxx?session_id=xxx"
}
d
```

## Step 2: Retrieve Request Results

```shell Get the completed request results
curl --location 'https://app.inklink.com/api/request-result/req_xxx' \\
--header 'x-api-key: secret_test_xxxx'
```

### Retrieve Session Results

```shell Get the results of a specific user
curl --location 'https://app.inklink.com/api/request-result/req_xxx?session_id=sid_xxx' \\
--header 'x-api-key: secret_test_xxxx'
```
